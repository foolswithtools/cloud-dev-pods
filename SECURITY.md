# Security policy

## Reporting a vulnerability

**Do not** open a public GitHub issue for security vulnerabilities.

Use **[Private vulnerability reporting](https://github.com/foolswithtools/cloud-dev-pods/security/advisories/new)** on this repo. Maintainers will respond within 7 days.

Include:

- A clear description of the vulnerability.
- Steps to reproduce.
- The cloud-dev-pods version (commit SHA or tag).
- Any proof-of-concept code (please be respectful — don't exploit production AWS accounts).

## Threat model

cloud-dev-pods provisions AWS infrastructure in **users' own AWS accounts**. The upstream repo at `foolswithtools/cloud-dev-pods` does not have access to any user's AWS account. Each downstream fork creates its own isolated trust relationship via GitHub OIDC.

## Out of scope

- Vulnerabilities in our dependencies (CDK, openvscode-server, oauth2-proxy, AWS SDKs). Report those upstream.
- Security implications of users' own customizations in `infra/extensions.local.ts`.
- Misconfiguration in users' AWS accounts, GitHub OAuth Apps, or IAM roles outside of what our IaC creates.

## Scope

- Code in `infra/lib/**`, `runtime/**`, `scripts/**`.
- IAM policies generated by our CDK code.
- The pod-manager Lambda's input validation.
- The upstream-sync algorithm in `scripts/sync-upstream.ts`.
- Public-repo safety mechanisms (workflow guards, OIDC trust scoping).
